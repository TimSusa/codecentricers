Core Concepts

Codecentricers is a single-page Javascript application (SPA) written in AngularJS. Right now we're using AngularJS version 1.4.0.

The application is structured into multiple layers following the Model-View-Controller pattern. Objects in the data model are created, read, updated and deleted through resource objects that communicate with the backend API.

Resources are managed inside services which take care of state updates, caching, listing and querying resources. Services are singleton instances and they are the main authority over any resource in Codecentricers.

Views, or application screens, are rendered from HTML templates which are loaded on demand by Angular's template system. The connection between views and the internal data model is managed by controllers.

Controllers are specific for views, but in order to reuse functionality controllers can be mixed. To manipulate a resource a controller may use the resource functions, to create, list, query or destroy resources the controller would call a service.
